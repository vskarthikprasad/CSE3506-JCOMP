We are testing SMOTE in conjunction with randomized under-sampling (RUS) as it will cut down computation time significantly, and can lead to better test-set performance in ROC space than the normal imbalanced data.

SMOTE uses KNN to generate synthetic examples, and the default nearest neighbours is K = 5. Well be sticking to this default throughout the analysis.

The steps SMOTE takes to generate synthetic minority (fraud) samples are as follows:

  • Choose a minority case: X
  • Find K nearest neighbors (in euclidean distance, as default) of X: X1, , X5
  • Randomly choose one of these K minority neighbors: X4
  • Generate a random number between 0 and 1: i
  • Create a synthetic minority observation somewhere between the 2 points: X + i(X4 - X)
  • Repeat dup_size times for each fraud case in the dataset. e.g. for dup_size = 1, SMOTE adds 1 synthetic fraud case for each real fraud case
